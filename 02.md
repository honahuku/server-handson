## SSH を使って GitHub の認証が出来ていることを確認
SSH を使って GitHub の認証が出来ていることを確認

VS Code などでターミナルを開いて以下のコマンドを入力
```bash
ssh -T git@GitHub.com
```
Hi honahuku! みたいな感じで自分のユーザー名が出てくれば成功です

![Image](https://GitHub.com/user-attachments/assets/73783007-cb6c-46dc-bd10-bbade6703ba5)

VS Code を使っている方はコマンドがエラーで終了した表示になっていることが気になるかもしれません
これはコマンドの応答に `but GitHub does not provide shell access.` とあるようにシェルを用いたアクセスが出来ないよ、ということを伝えるためにエラー終了しているだけで、 `You've successfully authenticated` と出ていれば問題なく SSH を使って GitHub への操作を行うことが出来ます。

SSH はシェルアクセスに用いられることが多いため、こういった通知をしているのではないかと考えられます

## docker で Go の helloworld を動かす
docker で Go の helloworld を動かすために、まず Go の開発環境を整えましょう

### Go の install

[Download and install - The Go Programming Language](https://go.dev/doc/install)

にある内容を確認し実施してください


- 手元の pc に Go の実行環境を整備し、 helloworld を書く
    - 以下の内容をを作業中のディレクトリに main.go として保存する
```go
// hello world の例
// (公式サイト https://go.dev/ にある実装と同じ) 
package main

import "fmt"

func main() {
	fmt.Println("Hello, world!")
}
```
```shell
# hello world プログラムの実行
go run main.go
```


### docker の環境構築が出来ていることを確認

```
docker run hello-world
```
を実行し docker が問題なく動くか確認しましょう。だめなら docker desktop のインストールなどを試してみてください


### Go のソースコードをビルドし、ビルド後のバイナリを実行するような image がビルドされる Dockerfile を書く
- [FROM](https://docs.docker.com/reference/dockerfile/#from) と [CMD](https://docs.docker.com/reference/dockerfile/#cmd) と COPY は最低限定義する必要があります
- Golang はオフィシャルイメージが出てます
    -  https://hub.docker.com/_/golang
    -  `golang:alpine` がおすすめです

- `docker build -t 任意のイメージ名 .` のあとに `docker run ビルドしたimageの名前` とすると `hello world!` みたいな感じで print した結果が表示される
